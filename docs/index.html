Give AI the whole IDE, not just the files.

v0.89.0

MCP Steroid is a Model Context Protocol server for JetBrains IDEs. It exposes IntelliJ Platform APIs, visual state, and runtime environment to any MCP-compatible AI agent via Kotlin code execution and screenshot capture.

The Problem

AI coding tools only read and write files. They cannot see or control the IDE, run refactorings, or debug with real runtime context.

The Solution

MCP Steroid connects any MCP client to IntelliJ-based IDEs so agents can use real IDE capabilities, not just files.

Capabilities

  • Visual understanding: screenshot capture, UI interaction, modal dialog detection
  • Full IDE control: refactorings, inspections, debugger actions, run tests
  • IntelliJ Platform access: PSI and project model queries via Kotlin code execution
  • Human review for actions before execution
  • Standard MCP protocol: no vendor lock-in

Integrates with any private or public IntelliJ Platform plugins from any vendor. Works with any MCP-compatible client:

Live IntelliJ Access

MCP Steroid runs inside IntelliJ, so agents execute Kotlin against real IDE APIs.

  • Standalone MCP server in the plugin exposes JSON-RPC over `/mcp` for each IDE session.
  • Kotlin suspend bodies run in IntelliJ JVM with PSI, VFS, refactoring, and inspection APIs.
  • Compile-then-run execution with review modes and append-only logs keeps agent work trackable.
